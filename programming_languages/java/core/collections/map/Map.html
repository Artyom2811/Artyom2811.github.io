Map

Основные разделы:

  • HashMap
  • TreeMap
  • LinkedHashMap - Используется если важен порядок добавления.
  • HashTable - HashMap с синхронизацией.
  • ConcurrentHashMap - потокобезопасный HashMap. С эффективной блокировкой.
  • ConcurrentSkipListMap - потокобезопасная версия TreeMap.
<р2>Ключ должен:
  1. Быть immutable
  2. Переопределить equals и hashcode
<р2>Вопросы на собеседование:
  1. Различия между HashMap и ConcurrentHashMap: - ConcurrentHashMap имеет locks каждого для bucket. При блокировке одного bucket, остальное не блокируются.
    1. имеет схожий интерфейс взаимодействия.
    2. в ConcurrentHashMap операции чтения не требуют блокировок и выполняются параллельно.
    3. в ConcurrentHashMap операции записи зачастую также могут выполняться параллельно без блокировок.
    4. в ConcurrentHashMap при создании указывается требуемый concurrencyLevel, определяемый по статистике чтения и записи.
    5. в ConcurrentHashMap value, объявленное как volatile.
  2. Hashtable и ConcurrentHashMap - Hashtable при обращении блокировалась полностью.
  3. HashTable и Collection.synchronized(HashMap) - Очень похожи. Но HashTable имеет функционал Enumerators(счетчики).
