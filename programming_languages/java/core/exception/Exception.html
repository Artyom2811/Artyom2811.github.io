Два главных типа:

  1. checked - компилятор их проверяет.
  2. unchecked - компилятор их не проверяет.

Основные определения:

  1. Exception(Исключение) - это нештатная ситуация, ошибка во время выполнения программы.
  2. Error - не обрабатываемые исключения тк после из возникновений нет гарантий что программа будет выполняться корректно.

Иерархия исключений:

Способы обработки исключений

  1. throws - ключевое слово которое, используется при определении метода. Указывает что метод может выкинуть ошибку.
  2. try - catch - finally 
    try(try-with-resources){}
catch(Exception e) {}
catch(ClassCastException | ArrayStoreException e1) {}
finally {}

    try-with-resources - дает возможность объявлять один или несколько ресурсов в блоке try, которые будут закрыты автоматически без использования finally блока. Проверяет наличие интерфейса AutoCloseable или Closeable. И выполняет метод close. 

    try (BufferedReader br = new BufferedReader(new FileReader(path))) {
    return br.readLine();
}

Вопросы на собеседование:

  1. Если наследовать класс в throws. Наследник должен иметь throws не выше по иерархии чем у родителя.
  2. Если используются несколько catch нужно писать их отлавливаемые исключения в порядке возвышения по иерархии.
  3. Можно написать try - finally без блока catch.
  4. try-with-resources - объект должен реализовывать Interface Closeable..
  5. Условия не выполнения finally:
    1. System.exit(int)
    2. Сбой в JVM
    3. Exception в finally
    4. Death of Daemon thread
