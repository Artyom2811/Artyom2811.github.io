Сериализация — это процесс сохранения состояния объекта в последовательность байт. Десериализация — это процесс восстановления объекта из этих байт.

  • Serializable интерфейс-маркер который мы должны имплементировать для сериализации объекта.
  • transient - ключевое слово используется на полях которые мы не хотим сериализировать.
  • long serialVersionUID - переменная необходимая для указания версии нашего класса. Она задается автоматически(не рекомендуется) или вручную. Пример мы сделали сериализацию, изменили класс и делаем десереализацию в уже измененый класс - нам вернется исключение java.io.InvalidClassException. Необходимо при изменения класса не забываем перегенерировать с помощью ide версию класса.
  • При сериализации все зависимости объекта должны быть тоже сериализуемы тк. они тоже сериализуются или получим ошибку. Или нужно объявить их transient.
