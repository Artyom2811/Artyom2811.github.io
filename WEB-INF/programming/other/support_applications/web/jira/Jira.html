Jira

Основные сущности

  1. Epic - большая предстоящая работа. Которая может быть разбита на множество Stories.
  2. Story(User stories) - описание полезности для пользователя будущего добавления.
  3. Sub-task - описание задач для реализации Story. Research, validation, implementation. Может быть зависима от другой Sub-task.
  4. Task - тоже что и Sub-task, но не связано с Story.
  5. Bug - исправление уже имеющейся ошибки.
  6. Spike - ознакомление и пробы библиотек или технологий, которые могут использоваться в последующих задачах.
