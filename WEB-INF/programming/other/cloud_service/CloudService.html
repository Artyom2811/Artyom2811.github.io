Облачный сервис

Проработать:

  1. Microsoft Azure - облачная платформа компании Microsoft. Предоставляет возможность разработки, выполнения приложений и хранения данных на серверах, расположенных в распределённых дата-центрах.
  2. Azure Databricks - платформа аналитики данных.

Основные разделы

  • AWS
  • Microsoft Azure
  • Google Cloud Platform(GCP)

Облачный сервис - это модель предоставления удобного сетевого доступа в режиме «по требованию» к коллективно используемому набору настраиваемых вычислительных ресурсов (например, сетей, серверов, хранилищ данных, приложений и/или сервисов), которые пользователь может оперативно задействовать под свои задачи и высвобождать при сведении к минимуму числа взаимодействий с поставщиком услуги или собственных управленческих усилий.

Модели облачных услуг

Выделяют 4 наиболее распространенных модели облачных услуг. Которые мы можем арендовать когда нам нужно:

Infrastructure as a Service (IaaS)

Виртуальные сервера, базы данных, компоненты сети, хранилище.

  • AWS EC2
  • Google Cloud
  • IBM Cloud
  • Microsoft Azure

Database as a Service (DBaaS) - под тип IaaS.

  1. AWS RDS
  2. Microsoft Azure Database
  3. Google Cloud SQL
  4. Heroku Postgres

Platform as a Service (PaaS) — платформа как услуга.

Вы занимаетесь только кодом, настройка инфраструктуры автоматическая.

  • AWS Elastic Beanstalk
  • AWS App Runner
  • AWS Lambda
  • Google App Engine
  • Google Cloud Functions
  • Microsoft Windows Azure
  • Red Hat OpenShift on IBM Cloud

Software as a Service (SaaS) — программное обеспечение как услуга.

Вы занимаетесь только использованием, ничего не нужно настраивать.

  • Gmail
  • Dropbox
  • Google Doc
  • Social media
  • Cloud file storage solutions (such as Dropbox or Box)
  • Trello
  • Slack
