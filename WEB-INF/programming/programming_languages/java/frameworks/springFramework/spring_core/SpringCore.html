Виды Конфигурации beans

  1. Конфигурация через аннотации с указанием пакета для сканирования — AnnotationConfigApplicationContext(“package.name”).
  2. Xml конфигурация — ClassPathXmlApplicationContext(“context.xml”).
  3. JavaConfig — конфигурация через аннотации с указанием класса (или массива классов) помеченного аннотацией @Configuration - AnnotationConfigApplicationContext(JavaConfig.class). Это как XML только в JAVA, без @Service и @Inject и тд., только @Bean. Используется когда мы используем сторонние библиотеки и не можем использовать на них аннотации.
  4. Groovy конфигурация — GenericGroovyApplicationContext(“context.groovy”).

Виды Dependency injection(DI) в порядке выполнения JAVA:

  1. Через конструкторы - лучший способ. Поля final, предотвращает Circular Dependency.
  2. Через поля - плохая практика.
  3. Через сеттеры

Без @Autowired

Начиная с Spring Framework 4.3 если внедряемый класс через конструктор, имея один конструктор, @Autowired можно не использовать.
