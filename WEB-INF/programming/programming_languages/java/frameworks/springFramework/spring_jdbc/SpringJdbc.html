  • Виды Конфигурации beans

Виды Конфигурации beans

  1. Конфигурация через аннотации с указанием пакета для сканирования — AnnotationConfigApplicationContext(“package.name”)
  2. Xml конфигурация — ClassPathXmlApplicationContext(“context.xml”)
  3. JavaConfig — конфигурация через аннотации с указанием класса (или массива классов) помеченного аннотацией @Configuration -AnnotationConfigApplicationContext(JavaConfig.class). Это как XML только в JAVA, без @Service и @Inject и тд. только @Bean.
  4. Groovy конфигурация — GenericGroovyApplicationContext(“context.groovy”)

Виды Dependency injection(DI):

  1. Через сэттеры
  2. Через конструкторы
  3. Через поля - плохая практика.

Без @Autowired

Начиная с Spring Framework 4.3 если внедряемый класс имеет один конструктор, @Autowired можно не использовать.
