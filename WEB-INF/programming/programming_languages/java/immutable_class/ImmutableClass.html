  1. Неизменное в Java: final, константы и Immutable

Immutable Class - класс который невозможно изменить после инициализации. Все Immutable сlass являются потокобезопасными(thread-safe) и не требуют синхронизации.

  1. Объявите класс как final, для предотвращения наследования, чтобы он не мог быть расширен.
  2. Сделайте все mutable поля final, чтобы их значение могло быть присвоено только один раз.
  3. Инициализируйте все поля через конструктор, выполняющий глубокое копирование.
  4. Выполните клонирование объектов в геттерах.

Immutable collection

Collections.unmodifiableList(java.util.List|<? extends T>>)
