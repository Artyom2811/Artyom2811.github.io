Порядок выполнения

  • Android читает AndroidManifest.xml
  • Ищет Activity с: 
    <application>
        ...>
    <activity
            android:name=".MainActivity">
        <intent-filter>
            <action android:name="android.intent.action.MAIN" />
            <category android:name="android.intent.category.LAUNCHER" />
        </intent-filter>
    </activity>
</application>
  • Создаёт эту Activity (например, MainActivity).
  • Вызывает её жизненный цикл:
    • onCreate()
    • onStart()
    • onResume() - обновление при возврате на страницу.

Элементы на UI

RecyclerView - который используется для отображения списков данных.

Обычный список (ListView) устарел и работает менее эффективно.

он “пересоздаёт” (recycles) элементы списка.

  • У тебя 1000 заметок
  • На экране видно только 10
  • Остальные 990 не создаются сразу
  • При прокрутке старые элементы переиспользуются

RecyclerView

У тебя список из 100 заметок, но экран помещает только 8. RecyclerView создаст примерно 8–10 ViewHolder, а потом будет переиспользовать их при прокрутке. Поэтому и называется RecyclerView — он перерабатывает (recycle) элементы.

  • onCreateViewHolder() - выполняется когда RecyclerView нужен новый элемент на экране.
  • onBindViewHolder() - заполняет View данными.
  • getItemCount() - говорит сколько элементов.
