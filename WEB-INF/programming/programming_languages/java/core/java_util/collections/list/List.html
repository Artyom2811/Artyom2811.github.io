List

Особенности List(список)

  • Элементы хранится по индексам.
  • Допускает дубликаты.
  • При добавлении примитивов происходит autoboxing.

Основные реализации

  1. ArrayList
  2. LinkedList
  3. Vector
  4. CopyOnWriteArrayList
  5. ConcurrentLinkedQueue - потокобезопасная очередь.

ArrayList

ArrayList - работает медленнее чем массив, облегчая работу с размером массива.

При добавлении в середину списка создаётся новый массив в него помещается: первая часть -> новый элемент -> вторая часть.

Если не хватает места - массив увеличивается на 1.5, но назад размер массива астоматически не уменьшается.

Сложность алгоритмов ArrayList

  1. Получение по индексу - O(1)
  2. Поиск по значению(contains) - O(n)
  3. Вставить в середину - O(n)
  4. Вставить конец - O(1), если не будет разширения.
  5. Удаление - O(n)

Оптимизация ArrayList

  • Указывайте ёмкость при создании - Если вы знаете (или можете оценить) количество элементов заранее - выделите память сразу.
  • Не используйте remove() в цикле - лучше .removeIf() оптимизирован под внутренние операции и работает быстрее при массовом удалении.
  • Будьте осторожны с автобоксингом: При работе с примитивными типами, такими как int, ArrayList автоматически упаковывает их в Integer. Это может привести к лишним затратам памяти. Если производительность критична, используйте специализированные классы, такие как IntArrayList из библиотеки fastutil.
  • Используйте метод trimToSize(): Если ваш ArrayList больше не будет изменяться, вызов метода trimToSize() освободит неиспользуемую память, оптимизируя использование ресурсов.

LinkedList

Реализация интерфейса List и Deque на основе двусвязного списка.

Элементы хранятся в объекте Node. Которые имеют ссылки на предыдущий и следующий элементы.

Рекомендуется использовать, если нужно вставлять элементы в начало или конец списка.

Каждый узел требует ~24 байта overhead (на 64-bit JVM)

Сложность алгоритмов LinkedList:

  1. Получение по индексу - O(n)
  2. Поиск из начала и конца - O(1)
  3. Поиск по значению(contains) - O(n)
  4. Вставить - в начало или конец - O(1), в середину по индексу - O(n), по значению - O(n).
  5. Удаление - из начало или конец - O(1), в середину по индексу - O(1), по значению - O(n).

CopyOnWriteArrayList

CopyOnWriteArrayList - потокобезопасная реализация List из пакета java.util.concurrent.

Все мутативные операции, такие как add, set, remove, clear, создают новую копию массива - работает с ней, при окончании работы выкидывается и при следующем обращении.

Внутри CopyOnWriteArrayList хранит данные в обычном массиве transient volatile Object[] array.

Итератор у CopyOnWriteArrayList не fail-fast, в отличие от ArrayList и LinkedList.

Что происходит при добавлении элемента(вызове add())?

  • Захватывается ReentrantLock - чтобы только один поток мог модифицировать список.
  • Создаётся новая копия внутреннего массива.
  • В копию добавляется новый элемент.
  • Volatile ссылка на внутренний массив атомарно заменяется на новый.
  • Lock освобождается

Чтение очень быстрое (без блокировок и синхронизаций).

Добавление — дорогое по памяти и CPU, потому что каждый раз копируется весь массив.

Когда использовать?

  • Чтений намного больше, чем записей
  • Список маленький или изменения редкие
  • Нужно безопасно итерироваться без блокировок

Не подходит:

  • Если много добавлений/удалений
  • Если список большой

Быстрый способ создания immutable collections(since v.9).

После создания никак нельзя изменить(add, remove, put). Но элементы в коллекции могут быть изменены. Нельзя добавлять null(NullPointerException).

  1. List.of("Luke", "Eva", "John");
  2. Map.of("name", "Luke", "age", "23", "city", "New York");
  3. Set.of("Luke", "Eva", "John");
