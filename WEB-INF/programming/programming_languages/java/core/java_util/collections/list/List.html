Особенности List(список)

  • Элементы хранится по индексам.
  • Допускает дубликаты.
  • При добавлении примитивов происходит autoboxing.

Основные реализации

  1. ArrayList
  2. LinkedList
  3. Vector
  4. CopyOnWriteArrayList - это потокобезопасный вариант of ArrayList. Разница в том, что все мутативные операции, такие как add, set, remove, clear, создают новую копию массива - работает с ней, при окончании работы выкидывается и при следующем обращении.
  5. ConcurrentLinkedQueue - потокобезопасная очередь.

ArrayList

ArrayList - работает медленнее чем массив, облегчая работу с размером массива.

При добавлении в середину списка создаётся новый массив в него помещается: первая часть -> новый элемент -> вторая часть.

Сложность алгоритмов ArrayList

  1. Получение по индексу - O(1)
  2. Поиск по значению(contains) - O(n)
  3. Вставить в середину - O(n)
  4. Вставить конец - O(1), если не будет разширения.
  5. Удаление - O(n)

LinkedList

Элементы хранятся в объекте Node. Которые имеют ссылки на предыдущий и следующий элементы.

LinkedList - используется если необходим функционал Queue, Dequeue или Stack.

Рекомендуется использовать, если нужно вставлять элементы в начало или конец списка.

Хоть LinkedList и имеет индекс.

Сложность алгоритмов LinkedList:

  1. Получение по индексу - O(n)
  2. Поиск из начала и конца - O(1)
  3. Поиск по значению(contains) - O(n)
  4. Вставить - в начало или конец - O(1), в середину по индексу - O(n), по значению - O(n).
  5. Удаление - из начало или конец - O(1), в середину по индексу - O(1), по значению - O(n).

Быстрый способ создания immutable collections(since v.9).

После создания никак нельзя изменить(add, remove, put). Но элементы в коллекции могут быть изменены. Нельзя добавлять null(NullPointerException).

  1. List.of("Luke", "Eva", "John");
  2. Map.of("name", "Luke", "age", "23", "city", "New York");
  3. Set.of("Luke", "Eva", "John");
