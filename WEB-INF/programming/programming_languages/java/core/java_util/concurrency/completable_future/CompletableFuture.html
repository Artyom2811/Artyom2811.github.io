CompletableFuture

Что это

  • Класс для асинхронного программирования.
  • Это расширение Future
  • Имеет callback - сообщает о завершении.
  • Комбинировать несколько задач
  • Обрабатывать ошибки без try-catch
  • Строить цепочки зависимостей без использования блокирующих операций

Ограничения Future

  • Его нельзя завершить вручную.
  • Нельзя выполнять дальнейшие действия над результатом Future без блокирования.
  • Нельзя повесить функцию-колбэк, чтобы она срабатывала автоматически, как только станет доступен результат.
  • Невозможно выполнить множество Future один за другим.
  • Невозможно объединить несколько Future.
  • Нет обработки исключений.
  • С Future ты просто ждешь результат. С CompletableFuture ты описываешь, что должно произойти, когда результат появится.

Создание

По умолчанию - ForkJoinPool.commonPool()

Но можно задать свой:

CompletableFuture.supplyAsync(() -> "Hi", Executors.newFixedThreadPool(10));

Методы

.allOf(...) - можно дождаться завершения сразу нескольких асинхронных задач, без ручной возни с join() по каждой.

Запуск выполнения

supplyAsync(Supplier) - можно вернуть результат.

runAsync() - нельзя вернуть результат.

Синхронное ожидание (если нужно)

get() - вызывающий поток будет ждать, отмена асинхронности. checked exception

join() - вызывающий поток будет ждать, отмена асинхронности. unchecked exception

Отмена

complete() - завершить вручную. Все клиенты, ожидающие этот Future, получат указанный результат, а последующие вызовы completableFuture.complete() будут игнорироваться.

Пример создания асинхронной задачи

CompletableFuture<String> cf = CompletableFuture.supplyAsync(() -> {
    // Эмуляция долгой задачи
    try {
        Thread.sleep(1000);
    } catch (InterruptedException e) {
        throw new IllegalStateException(e);
    }
    return "Задача завершена!";
});

System.out.println(cf.join()); // Ожидание завершения и получение результата

Комбинирование нескольких задач

Часто нужно дождаться завершения нескольких задач и собрать их результаты.

С помощью thenCombine() можно комбинировать результаты нескольких асинхронных вычислений

CompletableFuture<Integer> future1 = CompletableFuture.supplyAsync(() -> 50);
CompletableFuture<Integer> future2 = CompletableFuture.supplyAsync(() -> 20);

CompletableFuture<Integer> result = future1.thenCombine(future2, Integer::sum);

System.out.println(result.join()); // 70

Обработка исключений - exceptionally()

CompletableFuture<Integer> future = CompletableFuture.supplyAsync(() -> {
    if (true) throw new RuntimeException("Ошибка!");
    return 42;
}).exceptionally(ex -> {
    System.out.println("Произошла ошибка: " + ex.getMessage());
    return 0; // Возвращаем дефолтное значение в случае ошибки
});

System.out.println(future.join()); // 0

Обработка через handle()

Для восстановления после исключений API также предоставляет более общий метод handle(). Он вызывается независимо от того, возникло исключение или нет. 

Integer age = -1;

CompletableFuture<String> maturityFuture = CompletableFuture.supplyAsync(() -> {
    if (age < 0) {
        throw new IllegalArgumentException("Возраст не может быть отрицательным");
    }
    if (age > 18) {
        return "Взрослый";
    } else {
        return "Ребёнок";
    }
}).handle((res, ex) -> {
    if (ex != null) {
        System.out.println("Ой! У нас тут исключение - " + ex.getMessage());
        return "Неизвестно!";
    }
    return res;
});

System.out.println("Зрелость: " + maturityFuture.get());

Если возникает исключение, аргумент res будет null, если не возникает, то ex будет null.

Композиция цепочек

Методы

.thenApply() - аналог map(). Для обработки и преобразования результата CompletableFuture при его поступлении. Можно использовать цепочки thenApply. Может выполниться в том же потоке.

.thenApplyAsync() - точно в другом потоке (через пул).

.thenAccept() - не хотите возвращать результат, а хотите просто выполнить часть кода после завершения Future. Может выполниться в том же потоке.

.thenRun() - принимает Runnable и возвращает CompletableFuture<Void>

Каждая следующая задача запускается после завершения предыдущей, что позволяет гибко управлять зависимостями.

CompletableFuture.supplyAsync(() -> "Hello")
    .thenApply(result -> result + " World")
    .thenAccept(finalResult -> System.out.println(finalResult)); // Hello World

thenCompose() - аналог flatMap

allOf() - CompletableFuture.allOf(f1, f2, f3); - ждёт все

anyOf() - CompletableFuture.anyOf(f1, f2, f3); - ждёт первый завершившийся.

Применение на практике

Идеально подходит для задач, требующих асинхронности: запросы к API, обработка данных в фоне и т.д.

Пример с вызовом нескольких API параллельно:

CompletableFuture<String> user =
    CompletableFuture.supplyAsync(() -> getUser());

CompletableFuture<String> orders =
    CompletableFuture.supplyAsync(() -> getOrders());

CompletableFuture<String> result =
    user.thenCombine(orders, (u, o) -> u + " " + o);

System.out.println(result.join());
